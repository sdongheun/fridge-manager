# Fridge Manager MVP

## 목표

- 사용자가 자신의 냉장고를 계정 단위로 관리하고, 식재료를 등록한 뒤 유통기한을 빠르게 확인할 수 있는 모바일 앱을 만든다.
- 사용자는 카메라를 이용해 상품 정보와 유통기한을 관리할 수 있어야 하며, 수기 입력은 필요한 경우에만 최소한으로 사용해야 한다.

## 포함 범위

- 회원가입 기능을 제공한다.
- 로그인 및 로그아웃 기능을 제공한다.
- 사용자별 냉장고 데이터를 관리한다.
- 저장된 식재료 목록을 보여준다.
- 바코드 스캔 또는 수기 입력으로 식재료를 등록할 수 있다.
- 카메라를 이용해 상품 정보와 유통기한 입력을 보조할 수 있어야 한다.
- 수기 입력은 자동 인식이 실패하거나 사용자가 직접 보정이 필요할 때만 최소한으로 사용한다.
- 기존 식재료를 수정할 수 있다.
- 기존 식재료를 삭제할 수 있다.
- 유통기한이 임박한 항목이 먼저 보이도록 정렬한다.
- 신선 / 임박 / 만료 상태를 한눈에 구분할 수 있게 표시한다.
- 임박한 식재료에 대해 알림을 보낼 수 있다.
- 식재료 데이터는 DB에 저장되고 사용자 계정과 연결된다.

## 제외 범위

- 장바구니 기능
- 초대를 통한 공동 사용자 기능
- 냉장고 재료 기반 음식 추천 기능
- 복잡한 통계 기능
- 이미지 업로드 기반 식재료 등록
- 고급 관리자 기능

## 핵심 사용자 흐름

1. 사용자가 회원가입 또는 로그인을 통해 자신의 계정으로 진입한다.
2. 사용자가 자신의 냉장고 식재료 목록을 확인한다.
3. 사용자가 바코드 또는 카메라를 이용해 식재료 정보와 유통기한을 최대한 자동으로 등록한다.
4. 자동 인식이 부족한 경우에만 필요한 값만 짧게 수기 보정한다.
5. 사용자는 목록에서 유통기한 상태를 한눈에 확인한다.
6. 사용자는 필요 시 식재료를 수정하거나 삭제한다.
7. 사용자는 임박한 식재료에 대한 알림을 받는다.

## 성공 기준

- 사용자가 회원가입과 로그인을 통해 자신의 냉장고를 구분해서 사용할 수 있다.
- 사용자가 앱 안에서 식재료를 추가, 수정, 삭제, 조회할 수 있다.
- 바코드 스캔과 수기 입력 중 최소 하나는 안정적으로 동작하고, 최종적으로 둘 다 지원한다.
- 카메라 기반 입력 보조를 통해 사용자가 상품 정보와 유통기한을 더 적은 수기 입력으로 등록할 수 있다.
- 자동 인식이 실패하더라도 사용자는 필요한 값만 빠르게 보정해 등록을 끝낼 수 있다.
- 유통기한 상태를 한눈에 이해할 수 있다.
- 임박한 식재료 알림이 동작한다.
- 사용자 데이터는 DB에 안전하게 저장되고, 다른 사용자의 데이터와 분리된다.

## 현재 기술 가정

- 앱은 React Native 기반으로 개발한다.
- 백엔드는 Supabase를 사용한다.
- UI 스타일링은 Tailwind CSS 계열 방식을 사용한다.
- 패키지 관리는 `pnpm`을 사용해 저장 공간을 효율적으로 관리한다.
- 알림은 Expo의 알림 기능을 우선 검토한다.
- 현재는 MVP를 먼저 완성하고, 이후 협업/추천 기능을 확장한다.
- 코드 스타일은 typescript를 이용해 객체지향적으로 작성한다.

## 권장 기술 구성

- React Native 개발 환경은 현재 프로젝트 흐름상 Expo 기반을 유지하는 편이 구현 속도에 유리하다.
- Tailwind CSS 방식은 React Native에서 직접 CSS를 쓰는 것이 아니라 `NativeWind` 같은 호환 레이어를 사용하는 방식이 적합하다.
- Supabase Auth와 DB를 함께 사용하고, 사용자별 데이터 분리는 RLS(Row Level Security)를 기준으로 설계하는 것이 좋다.
- 알림은 Expo Notifications 기반으로 시작하고, 실제 운영 단계에서 로컬 알림과 푸시 알림의 역할을 나눠 설계한다.

## 추가 추천 항목

- `TypeScript strict mode`: 데이터와 화면 흐름이 늘어나기 전에 타입 안정성을 높이는 데 유리하다.
- `Zod`: 회원가입 입력값, 식재료 입력값, Supabase 응답 검증에 유용하다.
- `TanStack Query`: 서버 상태 조회, 캐시, 재요청 제어가 필요해질 때 Supabase 데이터 처리에 잘 맞는다.
- `React Hook Form`: 회원가입, 로그인, 식재료 등록 폼 관리에 효율적이다.
- `Expo Router`: 화면 수가 늘어나면 라우팅 구조를 정리하는 데 도움이 된다.

## 이후 추가할 기능

- 장바구니 기능: 마트에서 살 물품을 정리하는 노트
- 초대를 통한 공동 사용자 기능
- 냉장고에 있는 재료로 만들 수 있는 음식 추천 기능

## 사용자 시나리오

1. 사용자는 로그인을 통해 각자만의 냉장고를 관리할 수 있어야 한다.
2. 사용자는 바코드 또는 수기 입력으로 식재료를 등록하고 유통기한을 한눈에 파악할 수 있어야 한다.
3. 사용자는 식재료를 추가, 수정, 삭제할 수 있어야 한다.
4. 사용자는 임박한 식재료에 대한 알림을 받을 수 있어야 한다.
